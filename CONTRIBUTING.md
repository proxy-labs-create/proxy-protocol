# Contributing to PROXY Protocol

Thank you for your interest in PROXY. We are building in public and welcome contributions from the community.

---

## Ways to Contribute

### Protocol Design
The PROXY protocol specification is open for review and feedback. If you have suggestions on architecture, tokenomics, or the agent SDK design:

1. Open an **Issue** describing your suggestion
2. Label it `protocol-design`
3. We will review and respond

### Documentation
Improvements to documentation are always welcome:

1. Fork the repository
2. Make your changes
3. Submit a Pull Request with a clear description

### Bug Reports
If you find an issue with any published specification or documentation:

1. Open an **Issue**
2. Label it `bug`
3. Include as much detail as possible

### Provider Interest
If you are interested in running a PROXY provider node when testnet launches:

- Join the early access list at [proxyprotocol.org](https://proxyprotocol.org)
- We will notify you when provider registration opens

### Developer Interest
If you are a developer who wants to integrate the PROXY Agent SDK:

- Watch this repository for SDK alpha release
- Join the early access list at [proxyprotocol.org](https://proxyprotocol.org)

---

## Code of Conduct

- Be respectful and constructive
- No spam, self-promotion, or off-topic content
- Protocol criticism is welcome — personal attacks are not
